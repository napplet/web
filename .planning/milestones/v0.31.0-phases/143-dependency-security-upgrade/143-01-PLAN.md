# Plan 143-01: Upgrade Vulnerable Tooling Dependencies

## Objective

Resolve the `aislop` vulnerable-dependency findings for `vite`, `postcss`, and `turbo` without changing runtime behavior or forcing a Vite major migration.

## Tasks

<task id="143-01-01">
<title>Raise vulnerable dependency ranges</title>

<read_first>
- package.json
- packages/vite-plugin/package.json
</read_first>

<action>
Update the root `vite` range to `^6.4.2`, root `turbo` range to `^2.9.14`, and the package-local `@napplet/vite-plugin` Vite dev dependency to `^6.4.2`.
</action>

<acceptance_criteria>
- `package.json` contains `vite: ^6.4.2` and `turbo: ^2.9.14`.
- `packages/vite-plugin/package.json` contains `vite: ^6.4.2`.
</acceptance_criteria>
</task>

<task id="143-01-02">
<title>Refresh the pnpm lockfile</title>

<read_first>
- pnpm-lock.yaml
</read_first>

<action>
Run `pnpm install --lockfile-only` after manifest edits. If `postcss` remains below `8.5.10`, add the smallest package-manager override needed to force the patched transitive version, then refresh the lockfile again.
</action>

<acceptance_criteria>
- `pnpm-lock.yaml` resolves `vite >=6.4.2`.
- `pnpm-lock.yaml` resolves `postcss >=8.5.10`.
- `pnpm-lock.yaml` resolves `turbo >=2.9.14`.
</acceptance_criteria>
</task>

<task id="143-01-03">
<title>Verify dependency graph and security scanner</title>

<action>
Run `pnpm why vite`, `pnpm why postcss`, `pnpm why turbo`, and `pnpm dlx aislop scan --json .`. Save scanner JSON under `/tmp` and inspect vulnerable-dependency diagnostics.
</action>

<acceptance_criteria>
- `pnpm why vite` shows `6.4.2` or later.
- `pnpm why postcss` shows `8.5.10` or later.
- `pnpm why turbo` shows `2.9.14` or later.
- Scanner output has no vulnerable-dependency finding for `vite`, `postcss`, or `turbo`.
</acceptance_criteria>
</task>

<task id="143-01-04">
<title>Run workspace verification</title>

<action>
Run `pnpm -r type-check`, `pnpm -r build`, and `pnpm -r test:unit`.
</action>

<acceptance_criteria>
- All three workspace verification commands exit 0.
</acceptance_criteria>
</task>

## Verification

- `pnpm why vite`
- `pnpm why postcss`
- `pnpm why turbo`
- `pnpm dlx aislop scan --json .`
- `pnpm -r type-check`
- `pnpm -r build`
- `pnpm -r test:unit`

## Risks

- `postcss` may require an override if it is resolved transitively below the patched version.
- Vite minor updates may expose build-time typing changes in the Vite plugin package.

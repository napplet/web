# Plan 144-01: Remove Fixable Lint and AI-Slop Findings

## Objective

Remove the scanner's fixable lint and AI-slop findings while preserving TypeScript behavior and public runtime surfaces.

## Tasks

<task id="144-01-01">
<title>Run the mechanical scanner fixer</title>

<action>
Run `pnpm dlx aislop fix .`, then inspect the resulting diff. Keep only changes that delete/merge/remedy the scanner-reported fixable categories.
</action>

<acceptance_criteria>
- Mechanical changes are limited to scanner-targeted comments, duplicate imports, console leftovers, and equivalent fixable categories.
- `git diff --check` is clean after fixes.
</acceptance_criteria>
</task>

<task id="144-01-02">
<title>Manually close remaining lint findings</title>

<action>
Remove any remaining duplicate imports and unused type imports reported by the scanner.
</action>

<acceptance_criteria>
- No duplicate `@napplet/core` import diagnostics remain.
- The unused type imports named in the kickoff report are gone.
</acceptance_criteria>
</task>

<task id="144-01-03">
<title>Remove console leftovers and trivial/narrative comments</title>

<action>
Delete or replace leftover `console.log`, `console.debug`, and `console.info` calls in production source. Remove scanner-reported trivial and decorative comments while keeping useful API documentation and security rationale.
</action>

<acceptance_criteria>
- Scanner reports zero `ai-slop/console-leftover` diagnostics.
- Scanner reports zero `ai-slop/trivial-comment` diagnostics.
- Scanner narrative-comment count is zero or only explicit false positives documented in verification.
</acceptance_criteria>
</task>

<task id="144-01-04">
<title>Collapse duplicated central shim logic</title>

<action>
Refactor the duplicated block in `packages/shim/src/index.ts` into one local helper or a single shared path without changing global shim installation behavior.
</action>

<acceptance_criteria>
- Scanner no longer reports the duplicate-code block for `packages/shim/src/index.ts`.
- Existing shim installation behavior remains covered by type-check/build/unit tests.
</acceptance_criteria>
</task>

<task id="144-01-05">
<title>Verify cleanup</title>

<action>
Run the scanner, `pnpm -r type-check`, `pnpm -r build`, and `pnpm -r test:unit`.
</action>

<acceptance_criteria>
- Phase 144 scanner categories are closed or explicitly documented.
- Workspace type-check, build, and unit tests exit 0.
</acceptance_criteria>
</task>

## Verification

- `pnpm dlx aislop scan --json .`
- `pnpm -r type-check`

- `pnpm -r test:unit`

## Risks

- Removing public JSDoc indiscriminately could reduce package usability; keep useful API documentation even if it resembles a comment block.
- Console call deletion in the Vite plugin must not hide actual hard errors.

# Quick Task 260710-ng9: Fix napplet.run conformance pointer input

## Goal

Make the deployed conformance app accept canonical NIP-19 `nevent` and `naddr`
pointers for napplet manifest events instead of requiring an HTTP URL.

## Canonical Boundary

- NIP-5D napplets are manifest events of kind `5129`, `15129`, or `35129`.
- `naddr` is valid for addressable named napplet manifests.
- `nevent` is valid for concrete napplet manifest events, including snapshots.
- Relay hints from the pointer are the lookup source. If no relay hints exist,
  the UI must fail clearly instead of inventing a protocol discovery rule.

## Tasks

1. Add pointer resolution to `apps/conformance/src/main.ts`.
   - Decode `nevent`/`naddr` with the existing `nostr-tools` dependency.
   - Query relay hints for the manifest event.
   - Reject non-napplet event kinds.
   - Fetch `/index.html` from `server` tags and `path` hash data.
   - Feed the resolved HTML into the existing conformance runner.

2. Update the conformance UI and docs.
   - Replace URL-only copy, field type, query parameter naming, and README usage.
   - Preserve HTTP URL support only as a local/dev fallback for the CLI/live mode.

3. Verify.
   - Run focused type/build checks for `@napplet/conformance-web`.
   - Run root gates needed by this repo before commit/PR.

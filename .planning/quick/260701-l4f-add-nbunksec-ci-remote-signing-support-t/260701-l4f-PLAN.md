# Quick Task 260701-l4f: Add nbunksec CI remote signing support

**Date:** 2026-07-01
**Status:** complete

## Scope

Add nsyte-style CI signing for `@napplet/cli` by accepting `nbunksec` secrets through `--sec`, `NAPPLET_CI_SIGNING_KEY`, or `NAPPLET_CI_KEY_REFERENCE`/environment lookup and using NIP-46 remote signing for dry-run and network deploy events.

Out of scope: interactive bunker pairing/QR flow, storing/exporting nbunksec, new external dependencies, and profile/app/server-list publishing.

## Tasks

1. Convert the CLI signer abstraction to async.
   - Keep local hex/`nsec` signing working.
   - Await manifest signing and upload authorization signing.

2. Decode nsyte-compatible `nbunksec`.
   - Implement minimal bech32/TLV decoding locally to avoid a new dependency.
   - Validate pubkey, local key, and relay fields.

3. Add NIP-46 signer support.
   - Use `nostr-tools/nip46` `BunkerSigner` for `nbunksec`.
   - Close remote signer sessions after deploy/dry-run signing.
   - Keep `bunker://` explicitly unsupported unless a complete tested path is added.

4. Add tests and docs for CI signing behavior.

## Outcome

- Converted deploy signing to an async `NappletSigner` interface so local keys and remote NIP-46 signers share the dry-run, Blossom authorization, and relay publish paths.
- Added nsyte-compatible `nbunksec` TLV/bech32 decode/encode helpers without adding a new dependency.
- Wired `nbunksec` to `nostr-tools/nip46` `BunkerSigner`, including signer cleanup after dry-run or network deploy.
- Documented `NAPPLET_CI_SIGNING_KEY`/`NAPPLET_CI_KEY_REFERENCE` CI usage and kept raw `bunker://` pairing explicitly unsupported.
- Added a changeset for `@napplet/cli`.

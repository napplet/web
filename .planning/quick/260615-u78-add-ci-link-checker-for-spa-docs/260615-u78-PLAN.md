# Quick Task 260615-u78: CI link checker for SPA + docs

## Problem / Goal

The deployed site (napplet.run) is served from Bunny CDN storage + nsite, neither
of which rewrites extensionless paths to `.html`. A class of bug (VitePress
`cleanUrls: true` + SPA links to `/docs/packages/<name>`) shipped 404s that no
existing CI gate caught. Add a CI link checker that builds + assembles the site
exactly like `deploy-site.yml`, serves it the way the static host does (literal
files + directory `index.html`, **no** extensionless `.html` fallback), and
fails on any broken internal link.

The marketing SPA (`apps/web`) is **client-rendered** — its `/docs/...` links are
injected by Svelte at runtime and are absent from the static HTML. A static-HTML
link checker (linkinator/lychee/htmltest) therefore cannot see them. The crawler
MUST render pages in a real browser. This was verified locally: a static crawl
found only 4 links; a Playwright crawl found all 10 doc links.

The approach was proven locally end-to-end, including a negative test (reintroduce
the extensionless links → crawler reports the `/docs/packages/*` 404s and exits 1).

## Tasks

### Task 1 — `scripts/check-links.mjs` (Playwright BFS crawler)

- **files:** `scripts/check-links.mjs` (new)
- **action:** Create an ESM Node script that:
  - Imports `{ chromium } from 'playwright'`.
  - Base URL = `process.argv[2] || process.env.LINK_CHECK_BASE || 'http://localhost:8099'`.
  - BFS from `/`: for each same-origin URL, `fetch` it (follow redirects) to record
    status; if status >= 400 record as broken. If the response is `text/html` and
    not an asset, open it in the browser with `waitUntil: 'networkidle'` and collect
    `a[href]` to enqueue more internal links.
  - Normalization: resolve relative against the page URL; skip cross-origin
    (different `origin`), `mailto:`/`tel:`/`javascript:`/`data:`, and strip `#`
    fragments. Skip asset extensions (js, css, svg, png, etc.) for rendering but
    still status-check them.
  - On finish: print count checked; if any broken, print `[status] url` lines and
    `process.exit(1)`; else print success.
  - Wrap `page.goto` in try/catch so a render failure is reported, not thrown.
- **verify:** `node scripts/check-links.mjs http://localhost:8099` against the
  assembled+served site exits 0 with "0 broken".
- **done:** Script catches extensionless-link 404s (negative test) and passes clean.

### Task 2 — root `package.json`: playwright devDep + script + lockfile

- **files:** `package.json`, `pnpm-lock.yaml`
- **action:** Add `"playwright": "^1.59.1"` to `devDependencies`; add
  `"check:links": "node scripts/check-links.mjs"` to `scripts`. Run
  `pnpm install` to update `pnpm-lock.yaml` (CI uses `--frozen-lockfile`).
- **verify:** `pnpm exec playwright --version` works; lockfile updated.
- **done:** `pnpm check:links <url>` runs the crawler.

### Task 3 — `.github/workflows/link-check.yml`

- **files:** `.github/workflows/link-check.yml` (new)
- **action:** Mirror `deploy-site.yml` assembly. Triggers: `push`/`pull_request`
  to `main` with `paths` = `apps/web/**`, `apps/docs/**`, `packages/**`,
  `scripts/check-links.mjs`, `.github/workflows/link-check.yml`; plus
  `workflow_dispatch`. Steps: checkout → pnpm/action-setup@v4 →
  setup-node@v4 (node 22, cache pnpm) → `pnpm install --frozen-lockfile` →
  `pnpm build` → `pnpm --filter @napplet/web build` →
  `pnpm --filter @napplet/docs build` → assemble `./site` (web dist at root,
  docs dist at `site/docs`) → `pnpm exec playwright install --with-deps chromium`
  → start `python3 -m http.server 8099 --directory site &`, poll `curl` until it
  answers (bounded retries) → `node scripts/check-links.mjs http://localhost:8099`
  → kill the server.
- **verify:** YAML parses; mirrors deploy assembly.
- **done:** A PR touching the site runs the link check.

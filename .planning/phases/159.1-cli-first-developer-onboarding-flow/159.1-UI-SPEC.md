# Phase 159.1 - UI Design Contract

> Revised after maintainer review. This contract supersedes the executed
> six-step SPA layout while preserving the broader CLI-first workflow.

## Existing System

| Property | Value |
|----------|-------|
| Tool | Existing Svelte components and scoped CSS |
| Preset | Existing `apps/web/src/styles/global.css` tokens |
| Component library | None |
| Icon library | None; do not add one for this scoped surface |
| Fonts | Existing Inter/system sans and UI monospace stacks |

The phase changes `GetStarted.svelte` inside the existing SPA. It does not
redesign the site or introduce another visual language.

## Information Architecture

The section preserves the original boilerplate-era shape with this exact flow:

1. An important alpha warning overlays and blurs the onboarding content.
2. `I Understand` acknowledges the warning and unlocks the content.
3. Eyebrow: `Get started`.
4. Heading: `Install napplet in one command`.
5. Lead directs the developer to the visibly formatted `napplet guide` command.
6. One copyable command installs the CLI from `napplet.run/install.sh`.
7. Buttons link to docs, NIP-5D, community, and GitHub.
8. The persistent alpha note remains below the links.

The SPA does not display the create, init, skills, build, or deploy rows. Those
stages belong to `napplet guide`, where terminal output can keep them current
without making the page dense.

## Layout Contract

### Desktop and tablet

- Keep the existing constrained container and centered panel.
- Keep the warning card centered over the blurred panel content.
- Keep one inline install command between the lead and resource links.
- The command and copy label must remain distinct and must not overlap.

### Mobile and narrow screens

- Retain the same reading order and centered composition.
- At 560px and below, use the existing reduced panel and warning padding.
- The long installer URL wraps inside the command with
  `overflow-wrap: anywhere`.
- The copy label keeps a stable width while the command text wraps.
- Resource buttons wrap into readable rows without horizontal page overflow.

## Interaction Contract

### Alpha acknowledgement

- The initial state exposes a dialog with `aria-modal="true"` and a labelled
  heading.
- The underlying panel remains blurred, dimmed, inert, and hidden from
  accessibility navigation until acknowledgement.
- The warning says the CLI/tooling can be broken or incomplete and that the
  living specification can drift.
- Acknowledgement removes the overlay for the current page session only.

### Install command

- The surface contains exactly one command:

  ```text
  curl -fsSL https://napplet.run/install.sh | sh
  ```

- Clicking the command copies the full string.
- Feedback changes `copy` to `copied` for 1.6 seconds without resizing the
  control.
- Clipboard failure is non-critical and does not hide the command.

### Links

- Primary CTA: `Read the docs`.
- Secondary links: `NIP-5D spec`, `Group chat`, and `GitHub`.
- External links retain `target="_blank" rel="noopener"`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Warning kicker | `Alpha software` |
| Warning heading | `Before you install` |
| Warning action | `I Understand` |
| Heading | `Install napplet in one command` |
| Lead command | `napplet guide` |
| Installer | `curl -fsSL https://napplet.run/install.sh \| sh` |
| Primary CTA | `Read the docs` |

The installer must end successful interactive runs with bold colored guidance
to try `'napplet guide'`, visually distinguishing the command. The guide then
presents create, init, skills, verify, dry-run, and deploy commands in order,
with contextual docs URLs and a final documentation section.

## Hosted Assets

- `apps/web/public/install.sh` is byte-identical to
  `scripts/install-napplet-cli.sh`.
- `apps/web/public/install.ps1` is byte-identical to
  `scripts/install-napplet-cli.ps1`.
- Vite emits both files at the site root as `/install.sh` and `/install.ps1`.
- Root, package, and VitePress onboarding docs use the hosted URLs.

## Accessibility and Responsive Checks

- The warning dialog is visible and the underlying panel is inert before
  acknowledgement.
- The unlocked state has one copy button with the accessible name
  `Copy install command`.
- At 1440x900, 1024x768, 390x844, and 320x568, both warning and accepted states
  have no clipped command text, overlapping controls, or horizontal page scroll.
- The responsive navigation remains closed during the onboarding interaction.
- Warning and accepted screenshots are captured and inspected at every viewport.

## Registry Safety

No UI package, registry component, or new dependency is introduced.

# Phase 159.1 — UI Design Contract

> Visual and interaction contract for the CLI-first SPA onboarding surface.

## Existing System

| Property | Value |
|----------|-------|
| Tool | Existing Svelte components and scoped CSS |
| Preset | Existing `apps/web/src/styles/global.css` tokens |
| Component library | None |
| Icon library | None; do not add one for this scoped surface |
| Fonts | Existing Inter/system sans and UI monospace stacks |

The phase changes `GetStarted.svelte` inside the existing SPA. It does not
redesign the site, add a second visual language, or add decorative imagery.

## Information Architecture

The section is one unblocked surface with this exact reading order:

1. Eyebrow: `Get started`
2. Heading: `One path from install to deploy`
3. Lead explaining standalone CLI, maintained starter, skills, and deploy
4. Platform selector for the install command (`macOS`, `Linux`, `Windows`)
5. Ordered workflow:
   - Install the CLI
   - Create the project
   - Initialize deployment
   - Install agent skills
   - Build and verify
   - Preview and deploy
6. Docs/spec/community/GitHub links
7. Persistent alpha note without interaction blocking

No dialog, blur, lock state, hidden content, or acceptance button is permitted.

## Layout Contract

### Desktop (>= 900px)

- Keep the existing constrained container.
- The panel uses a two-column grid: intro/links at `minmax(240px, 0.72fr)` and
  ordered workflow at `minmax(0, 1.28fr)` with a 48px gap.
- The intro may become sticky only if it does not overlap the footer or section
  bounds; otherwise it remains static.
- The workflow is one ordered list, not six cards. Each row uses a 40px marker
  track plus `minmax(0, 1fr)` content and a top separator after the first row.

### Tablet/mobile (< 900px)

- Stack intro above workflow.
- At <= 560px, container/panel horizontal padding is 16-20px.
- Command rows use `grid-template-columns: minmax(0, 1fr) 40px`.
- Code wraps with `overflow-wrap: anywhere`; no horizontal viewport overflow.
- Platform segments wrap or use three equal tracks; labels never truncate.

### Stable dimensions

- Step marker: 40px square, fixed.
- Copy control: 40px square, fixed, with an accessible text label/tooltip.
- Platform selector: three equal columns where space permits.
- Dynamic `Copied` feedback must not resize the row; expose it through the
  existing fixed control and an `aria-live` status.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant | `var(--bg-soft)` / `var(--surface)` | Panel and command background |
| Secondary | `var(--border)` / `var(--text-muted)` | Separators and supporting copy |
| Primary accent | `var(--accent-bright)` | Selected platform, active copy affordance, heading accent |
| Supporting cyan | `var(--cyan)` | Install/create step marker accents only |
| Supporting amber | `var(--amber)` | Deploy/alpha note accents only |
| Text | `var(--text)` | Titles and commands |

No new gradient, orb, bokeh, or decorative background is added. The existing
page/panel treatment may remain. Accent is reserved for selected state, command
actions, and step markers; body copy remains neutral.

## Interaction Contract

### Platform selector

- Render as a segmented control with three buttons: `macOS`, `Linux`, `Windows`.
- Use `aria-pressed` or tab semantics consistently; keyboard activation must work.
- Selection changes only the install command. It does not reorder or hide steps.
- Default to `macOS` only as a static render default; client-side platform
  detection may select a better match but is not required.

### Copy controls

- Every command row is independently copyable.
- The whole command row may be clickable only if the nested copy control is not
  duplicated; prefer one explicit fixed copy button per row.
- Clipboard failure is non-blocking and leaves the command selectable.
- Feedback reads `Copied` through `aria-live` for 1.6 seconds and then resets.

### Links

- Primary CTA: `Open the getting-started guide`.
- Secondary links: `NIP-5D spec`, `Community`, and `GitHub`.
- External links retain `target="_blank" rel="noopener"`.

### Focus and motion

- Every new button receives a visible `:focus-visible` outline using accent.
- Existing reveal behavior may animate the section once; individual step rows do
  not cascade or move independently.
- Reduced-motion behavior inherits the global rule.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Heading | `One path from install to deploy` |
| Lead | `Install the standalone CLI, scaffold the maintained starter, add the napplet skills to your coding agent, and deploy when the build passes.` |
| Install step | `Install the CLI` |
| Create step | `Create the project` |
| Init step | `Initialize deployment` |
| Skills step | `Install agent skills` |
| Build step | `Build and verify` |
| Deploy step | `Preview and deploy` |
| Primary CTA | `Open the getting-started guide` |
| Alpha note | `napplet and its protocol are alpha. Review the generated manifest and dry-run output before publishing.` |
| Clipboard status | `Command copied` |

Avoid claims that the CLI itself writes application code. The agent builds the
napplet after skills installation; the CLI scaffolds, configures, verifies, and
deploys around that handoff.

## Command Contract

The surface uses real commands only:

```text
curl -fsSL https://raw.githubusercontent.com/napplet/napplet/main/scripts/install-napplet-cli.sh | sh
powershell -NoProfile -ExecutionPolicy Bypass -Command "irm https://raw.githubusercontent.com/napplet/napplet/main/scripts/install-napplet-cli.ps1 | iex"
napplet create my-napplet
cd my-napplet && napplet init
napplet skills install --to codex
pnpm verify
napplet deploy --dry-run
napplet deploy
```

The displayed skill target may use Codex as the concrete example while docs list
all supported targets.

## Accessibility and Responsive Checks

- Section heading labels the workflow region.
- Ordered list semantics preserve the six-step sequence.
- Platform state is conveyed programmatically, not by color alone.
- Copy buttons have command-specific accessible names.
- `aria-live="polite"` announces successful copies.
- At 1440x900, 1024x768, 390x844, and 320x568: no clipped command text,
  overlapping controls, horizontal page scroll, or hidden CTA.
- 200% zoom retains a readable sequence and reachable copy controls.

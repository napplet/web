<objective>
Implement NAP-SHELL — the mandatory, foundational bootstrap handshake — end to end across `@napplet/core`, `@napplet/nap`, `@napplet/shim`, and `@napplet/conformance`, replacing the old private `{ naps, sandbox }` `shell.init` shape with the canonical `{ capabilities: { domains, protocols }, services, class }` environment.

NAP-SHELL is the one NAP that cannot be discovered via `supports()`. The shim posts `shell.ready` (no payload, already done today — keep it), the runtime replies once with `shell.init`, the shim caches that environment, and `window.napplet.shell` answers `supports(domain, protocol?)` synchronously and locally thereafter. The public API also grows `services`, `class`, `ready()`, and `onReady()`. The conformance engine recognizes `shell.*` as first-class NAP-SHELL envelopes (dropping the reference-shell special-case), its reference shell replies in the new shape, and the boot/degrade checks cite NAP-SHELL as their contract.

Source of truth: `/tmp/NAP-SHELL2.md`. Never invent surface beyond it (AGENTS.md "Protocol fidelity").

Purpose: Align the SDK + conformance engine to the updated NAPs track so `shell` is a real foundational domain, not a private handshake special-cased through the validator.
Output: New `@napplet/core` shell types, a `@napplet/nap/shell` subpath, a migrated shim, a migrated conformance reference shell + validator + checks, updated tests, and updated docs. Green at every commit.
</objective>

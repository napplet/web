<objective>
Migrate first-party packages from the old generic shell bootstrap/capability
query to runtime-injected optional NAP domain objects.
</objective>

<tasks>
1. Remove active shell-handshake types and tests from `@napplet/core`; make
   `NappletGlobal` domain properties optional to model domain absence.
2. Refactor `@napplet/shim` into a runtime injection installer that can install
   selected NAP domains, registers message routing, and does not emit
   `shell.ready` or wait for `shell.init`.
3. Remove `@napplet/nap/shell` exports and shell SDK/shim helpers from the
   active package surface.
4. Update SDK/domain helper guards so absent domains produce clear errors and
   type examples use property-presence checks.
5. Update package unit tests around optional domains, selected injection, and
   no-handshake behavior.
</tasks>
